## 3. Архитектура RAG (что построено)

```
RAG_SOURCE_PATH/  (PDF, md, txt — только локально)
        ↓  tools/ingest_from_g.py  (.md / .txt / .pdf, pypdf)
05data/rag_index/  (ChromaDB, локально)
        ↓
02modules/dzen-rag/backend  (FastAPI)
        ↓
```

## 4. Реализация

### Стек

- **Backend:** FastAPI, ChromaDB, sentence-transformers, pypdf, httpx  
- **Embeddings:** `paraphrase-multilingual-MiniLM-L12-v2` (локально)  
- **LLM:** DeepSeek API (основной), LM Studio (опционально)  
- **Общий код:** `aikivaviora_shared/rag/`

### API-эндпоинты

| Метод | Путь | Назначение |
|-------|------|------------|
| GET | `/health` | Статус, `docs_count`, флаги LLM |
| POST | `/upload` | Загрузка PDF/TXT (методичка ДЗ-7) |
| POST | `/ingest` | Переиндексация с G: |
| POST | `/chat` | Вопрос (`query` / `question`) + ответ LLM |
| GET | `/ui/` | Загрузка документов + чат |

## 5. Тестирование

### Retrieval (поиск по базе)

**Вопрос:** «Что делегировать ИИ инженеру?»

- Найдены фрагменты из PDF урока 09:00 (Инженер, модуль 1.3)
- Score релевантности ~0.72–0.73
- Источники отображаются в `/chat` и во фронте `/ui/`

### Генерация ответа (DeepSeek)

- Провайдер: `auto` → `deepseek`
- `llm_status: ok`
- Ответ структурирован по контексту: жёлтая зона (можно делегировать) vs красная (нельзя)

### Пример вопросов для проверки (модуль 1.3)

1. Что делегировать ИИ инженеру?  
2. Где граница между помощью ребёнку и списыванием?  
3. Какие промпты использовать школьнику для учёбы?  
4. Какие задачи учитель может делегировать ИИ, а какие нет?

### Сложности

| Проблема | Решение |
|----------|---------|
| Порт 8000 занят Windows | Запуск на **8001** (`start-api.bat`, `run.ps1`) |
| uvicorn «Could not import app» | Запуск только из папки `backend/` |
| `no_llm_configured` при наличии ключа | Перезапуск API; порядок загрузки `.env` |
| PDF с эмодзи в имени файла | Ingest работает; в метаданных — полное имя файла |
