## 3. Архитектура RAG

```
RAG_SOURCE_PATH/     ← PDF, md, txt (только локально)
        ↓  ingest_from_g.py / POST /upload / POST /ingest
05data/rag_index/    ← ChromaDB (локально, не в git)
        ↓
02modules/dzen-rag/backend  (FastAPI :8001)
        ↓
DeepSeek API  |  LM Studio :1234
        ↓
frontend /ui/        ← пополнение базы + чат + справка
```

### Назначение продукта (для проверяющего)

- **Главное:** RAG-ассистент — вопросы по **уже проиндексированным** материалам канала, ответ с источниками.  
- **Загрузка файла:** пополнение векторной базы (чанки в Chroma), **не** конвертация PDF.  
- **Не делает:** публикацию в Дзен, перевод форматов, ответы без базы (off-topic → фраза из промпта).

### Отличия от TrailCamp (урок 7)

| TrailCamp | Dzen RAG |
|-----------|----------|
| Gemini embeddings + Flash | sentence-transformers + DeepSeek / LM Studio |
| POST `/upload` | ✅ + ingest пакетом |
| React :5173 | Статический `/ui/` :8001 |
| `chroma_db/` | `05data/rag_index/` |

## 4. Реализация

### Стек

FastAPI · ChromaDB · sentence-transformers · pypdf · httpx · `aikivaviora_shared/rag/`

### API

| Метод | Путь | Назначение |
|-------|------|------------|
| GET | `/health` | Статус, число чанков, LLM |
| POST | `/upload` | Один PDF/TXT/MD в базу (ДЗ-7) |
| POST | `/ingest` | Переиндексация папки из `RAG_SOURCE_PATH` |
| POST | `/chat` | `query` или `question` + ответ LLM |
| GET | `/ui/` | Веб-интерфейс |

**Retrieval:** «Что делегировать ИИ инженеру?» — фрагменты PDF 09:00, score ~0.72.
