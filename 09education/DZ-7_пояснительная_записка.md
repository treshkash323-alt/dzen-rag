## Аннотация

Разработан прототип RAG-ассистента для поиска и ответов по проиндексированным PDF и markdown материалам образовательного модуля Дзен-канала. Пайплайн включает ingest, векторное хранилище ChromaDB, REST API на FastAPI и веб-интерфейс. В качестве LLM использован DeepSeek API; эмбеддинги — локальная модель sentence-transformers.

---

## 1. Постановка задачи

### 1.1. Цель

Получить инструмент, который по **собственным** материалам канала (уроки, практики, статьи в PDF/md) отвечает на вопросы пользователя на русском языке с указанием источников, без опоры на «общие знания» модели вне базы.

### 1.2. Задачи

1. Организовать загрузку и индексацию документов (PDF, TXT, MD).  
2. Реализовать векторный поиск релевантных фрагментов (RAG retrieval).  
3. Подключить LLM для генерации ответа по найденному контексту.  
4. Предоставить API и веб-UI по требованиям методички (в т.ч. `POST /upload`, `POST /chat`).  
5. Обеспечить отказ при вопросах вне базы знаний.  
6. Подготовить проект к публичному размещению на GitHub без утечки секретов.

### 1.3. Границы проекта

- Не публикация в Дзен, не конвертация PDF в другие форматы.  
- Не замена общего ChatGPT — только документы, попавшие в индекс.  
- MVP для локального запуска (localhost); аутентификация API не реализована.

## 4. Реализация

### 4.1. Модули программы

| Модуль | Функция |
|--------|---------|
| `loaders.py` | Чтение PDF (pypdf), md, txt |
| `chunker.py` | Разбиение на чанки 900 / overlap 120 |
| `store.py` | ChromaDB, upsert, query, delete_by_filename |
| `ingest.py` | Пакетный ingest и upload одного файла |
| `llm.py` | Вызов OpenAI-compatible API, системный промпт |
| `app.py` | FastAPI-эндпоинты, CORS, лимиты безопасности |

### 4.2. API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/health` | Статус, `docs_count`, флаги LLM |
| POST | `/upload` | Multipart: один PDF/TXT/MD в индекс |
| POST | `/ingest` | Переиндексация каталога (внутри `RAG_SOURCE_PATH`) |
| POST | `/chat` | Поля `query` или `question`, `provider`, `top_k` |
| GET | `/ui/` | Статический интерфейс |

### 4.3. Веб-интерфейс

- Постоянный блок **назначения продукта** (чат vs пополнение базы).  
- Секции: пополнение базы, настройки LLM, чат.  
- Справка: «О программе», «Глоссарий», «Как пользоваться».  
- Защита от XSS: экранирование вывода в `app.js`.

### 4.4. Конфигурация

Файл `backend/.env` (не в git): `RAG_SOURCE_PATH`, `DEEPSEEK_API_KEY`, `RAG_LLM_DEFAULT`, параметры чанкинга. Пример: `config/paths.example.env`.

## 5. Результаты экспериментов

### 5.1. Ingest

- 18/18 файлов обработаны без ошибок.  
- 126 чанков в коллекции `dzen_rag`.

### 5.2. Retrieval

Тестовый запрос: *«Что делегировать ИИ инженеру?»*  
Результат: фрагменты из PDF урока 09:00 (роль Инженер), similarity score порядка **0.72**.

### 5.3. Генерация

- Провайдер: **DeepSeek**, `llm_status: ok`.  
- Ответ структурирован по контексту (делегируемое / неделегируемое).  
- В UI и JSON — список `sources` с именами файлов.

### 5.4. Off-topic

Запрос вне темы базы (например, рецепт борща) → ответ с формулировкой **«Информация не найдена в базе знаний»** или отсутствие релевантного контекста — без выдуманных фактов из модели.

### 5.5. Загрузка через API

`POST /upload` и форма на `/ui/` добавляют файл в индекс; повторная загрузка того же имени заменяет чанки файла.
