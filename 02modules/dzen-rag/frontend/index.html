Поиск и ответы по базе материалов Дзен-канала «Канал про ИИ»

загрузка…

Это RAG-ассистент, а не конвертер файлов: по вашим PDF и статьям канала ищет релевантные фрагменты и формулирует ответ с указанием источников.

  • Чат основной режим — вопросы по уже проиндексированным материалам модуля
  • Загрузка пополнение векторной базы: файл режется на чанки и добавляется в Chroma для поиска
  • Не делает не переводит PDF в другой формат и не публикует в Дзен — только индекс + Q&A

Dzen RAG — рабочий стол для ДЗ-7 и авторов AIKIVAVIORA

Для каких задач

  • Быстро вспомнить содержание модуля, урока или PDF канала «Канал про ИИ»
  • Задать вопрос по-русски и получить ответ только по проиндексированным текстам
  • Проверить цитату: под ответом раскрываются фрагменты-источники (файл + score)
  • Добавить новый PDF/TXT/MD в базу без ручной разметки (ingest через форму или API)

Чем отличаются блоки на экране

Чат (внизу)
Главная функция. Вы задаёте вопрос → поиск в Chroma → LLM (LM Studio / DeepSeek) собирает ответ из найденных чанков.
Загрузка документа
Вспомогательная: один файл уходит в векторную базу (POST /upload). Это пополнение знаний, не конвертация. Повтор той же книги заменяет её чанки.
Пакет с диска G:
Массовая индексация папки модуля — через API /ingest или скрипт (уже может быть сделана: счётчик чанков в бейдже).

Кому полезно

Авторам модулей Дзен, студентам курса VibeCoder, команде AIKIVAVIORA — когда нужен «умный поиск» по своим материалам, а не общий ChatGPT.

Dzen RAG — учебно-рабочий модуль экосистемы AIKIVAVIORA (филиал Cursor v.3, ДЗ-7).

  • Стек UI: HTML/CSS/JS, API FastAPI
  • Индекс: ChromaDB + локальные embeddings
  • LLM: LM Studio (локально) или DeepSeek (облако)

Курс VibeCoder · ДЗ-7 RAG · филиал Cursor v.3

RAG
Retrieval-Augmented Generation — сначала поиск фрагментов в базе, затем ответ LLM по ним.
Чанк
Кусок текста (~900 символов) для индекса; у каждого — источник (PDF/md).
Ingest
Загрузка материалов в Chroma: пакет с G: (/ingest) или файл (/upload).
Embeddings
Числовой «смысл» текста; похожие фрагменты находятся ближе в векторной базе.
top_k
Сколько фрагментов подтянуть в контекст перед ответом (1–20).
provider
auto — из .env; lm_studio — локально; deepseek — облако; none — только цитаты.
score
Близость фрагмента к вопросу (выше ≈ релевантнее).
Слот 1F–9F
Время публикации в сетке модуля Дзен (видео, уроки, практика по ролям).
  1. Убедитесь, что в бейдже есть чанки (база уже с G: или загрузите PDF/TXT ниже).
  2. Выберите LLM (lm_studio — если запущен LM Studio на :1234).
  3. В Чате задайте конкретный вопрос по модулю — это основной сценарий.
  4. Загрузка нужна только когда добавляете новый файл в базу, а не для каждого вопроса.
  5. Раскройте источники под ответом — сверьте с PDF при сомнениях.

Пополнение базы знаний

Опционально: добавить PDF, TXT или MD в векторную базу для поиска в чате (не конвертация). Повтор того же файла заменяет его чанки.

Чат — вопросы по базе

Основной режим: задайте вопрос, если в бейдже уже есть чанки. Загрузка выше — только для новых файлов.

Задайте вопрос по материалам канала. Ответ строится из фрагментов в Chroma; при пустой базе — сначала индекс (G: или блок «Пополнение базы»).
