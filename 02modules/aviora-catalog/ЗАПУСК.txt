AIKIVAVIORA Catalog — как пользоваться (без путаницы)
====================================================

1) ОДИН сервер на порту 8002 — выберите ОДИН способ:

   A) Двойной клик start-api.bat
      • Чёрное окно НЕ закрывать — это и есть сервер!
      • Закрыли окно = Catalog выключился.

   B) Терминал VS Code / Cursor (если удобнее IDE):
      cd ...\aviora-catalog\backend
      ..\stop-api.bat          (если порт занят старым процессом)
      .\run_uvicorn.ps1
      • Этот терминал не закрывать, пока работаете с Catalog.

   • НЕ запускайте bat И uvicorn в VS Code одновременно.
   • Ошибка 10048 = второй запуск; сначала stop-api.bat

2) Обновить страницу в браузере
   • Ctrl+Shift+R
   • Внизу страницы должно быть: v0.3.8 (не v0.3.7)

3) Индекс для поиска
   • Кнопка Scan → Да → ждите синюю полосу прогресса до конца
   • Внизу: «health ok · 3500+ files» (не 0 files)

4) Чат (Send)
   • LM Studio: модель загружена → Start Server (:1234)
   • В шапке Catalog: lmstudio, ON, зелёный «в сети»
   • Send — один раз, дождаться ответа

5) Поиск слева
   • Слово в поле «Поиск — Enter» → клавиша Enter (не Send)

ЧТО ЗАКРЫВАТЬ / ЧТО ОСТАВЛЯТЬ
----------------------------

Пока работаете с Catalog (браузер http://127.0.0.1:8002/ui/):
  ОСТАВИТЬ открытым:
    • окно терминала от start-api.bat (сервер :8002)
    • вкладку браузера с Catalog
  Можно закрыть:
    • лишние терминалы VS Code с uvicorn (если там была ошибка 10048)
    • LM Studio — только если чат не нужен; для Send нужен Start Server :1234

Когда закончили на сегодня:
  1) Закройте вкладку браузера Catalog (по желанию)
  2) Закройте окно start-api.bat или Ctrl+C в нём — сервер :8002 остановится
  3) LM Studio — по желанию (Stop Server экономит RAM)
  4) Cursor / VS Code — закрывайте как обычно, на Catalog не влияет

НЕ нужно:
  • останавливать Python_kash или «весь компьютер»
  • удалять .venv или .env
  • каждый раз делать Scan — индекс сохраняется (05data/aviora_catalog_index.json)

Перезапуск Catalog:
  stop-api.bat  →  start-api.bat
  (или stop-api.bat  →  run_uvicorn.ps1 в VS Code)

Почему «заработало» после закрытия bat:
  На :8002 был чужой/старый python; bat только открывал браузер.
  VS Code запустил правильный .venv — вы закрыли bat, остался VS Code.
  Теперь bat сам проверяет v0.3.8 и перезапускает старый процесс.
